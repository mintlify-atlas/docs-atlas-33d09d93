Kami (紙, かみ) is the Japanese word for paper — the quiet surface on which a finished idea finally lands. It is a document design system built for AI agents: one visual language across six document types, so every piece you generate looks like something a person made with care.

Warm parchment canvas, ink-blue as the sole accent, serif carries authority, no hard shadows or flashy palettes. Kami is part of a trilogy: [Kaku](https://github.com/tw93/Kaku) writes code, [Waza](https://github.com/tw93/Waza) drills habits, **Kami ships documents**.

## How to install

<Steps>
  <Step title="Add the Kami skill to your AI agent">
    Run the install command for your agent of choice. No configuration needed — the skill auto-triggers when you describe a document task.

    <CodeGroup>
    ```bash Claude Code
    npx skills add tw93/kami -a claude-code -g -y
    ```

    ```bash Codex
    npx skills add tw93/kami -a codex -g -y
    ```

    ```bash Generic agents
    npx skills add tw93/kami -a '*' -g -y
    ```
    </CodeGroup>
  </Step>
  <Step title="Describe what you need">
    Tell your agent what to create — in any language. Kami recognizes document tasks automatically.

    > make a one-pager for my startup / build me a resume / design a slide deck for my talk / turn this into a polished white paper
  </Step>
  <Step title="Build and verify">
    Use the build script to render PDFs and verify page counts, font embedding, and CSS rule compliance.

    ```bash
    python3 scripts/build.py --verify
    ```
  </Step>
</Steps>

<Note>
  Kami uses [WeasyPrint](https://weasyprint.org/) to render HTML templates to PDF. Make sure WeasyPrint and pypdf are installed before running `build.py`.
</Note>
